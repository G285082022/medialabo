「Webコンテンツ作成の成果」

本テーマの学習目的

本テーマの学習目的は以下の5つです。

  • HTML によって Web ページの構造を記述する基本的な方法がわかる
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる
  • HTML, CSS, JavaScript の構文リファレンスから、構文の書式や使用方法の情報を入手できる
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる

プロジェクト課題について

数当てゲーム

プロジェクト課題では「世界の天気」を実施しました。できるだけわかりやすく、シンプルになるよう心がけて制作しました。
Webページ全体に世界地図を表示し、サーバーから得られる全ての地点の天気を、世界地図の場所通りにアイコンで表示します。 ここで表示されるアイコンは空模様しかわからないので具体的な気温や湿度は各地点に表示されたアイコンをクリックすることで表示するようにしました。
こうすることで一目で世界各地の天気を知ることができます。また、細かい情報はアイコンをクリックするという直感的な操作で行えるため、わかりやすくやりやすい操作を実現できました。

毎回の実験で作成したページ

第1回 WWW と HTML

第1回はWebページとWWWについてと、HTMLを用いてWebページの構造を記述する基本的な方法を学びました。
WebページはHTML, CSS, JavaScript, メディアの4つの部品からなり、WWWによってサーバからWebブラウザにWebページの部品が送られます。送られた部品をWebブラウザが組み立て、Webページを表示します。
HTMLはWebページの構造を記述する言語で、さまざまな要素でつくられます。要素には大きく分けてブロック要素とインライン要素があります。ブロック要素は基本的な構造を決める要素で、インライン要素はブロック要素を修飾してより細かく構造を決める要素です。

演習の成果

  • 演習1-3: HTML文書の作成
  • 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回はCSSを用いてWebページの装飾を記述する基本的な方法を学びました。
CSSはHTMLファイルに適用することでWebページの要素を装飾することができます。今回は、外部スタイルシートのファイルを指定する方法で適応しました。CSSにはさまざまなプロパティがあり、各プロパティに応じたプロパティ値を設定してWebページを装飾します。

演習の成果

  • 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

第3回はJavaScriptを用いて動きのあるWebページを実現するための基礎的なことを学びました。
JavaScriptはCSSと同様にHTMLファイルに適用することでWebページに’動き’を追加することができます。今回は、CSSと同様に外部指定でHTMLファイルに適用しました。その際にdefer属性を指定しないとプログラムが正しく動作しないことがあります。
Javaと構文は似ていますが、Javaとは違いインタプリタ言語で1行ずつ読み取りながら実行されます。コンパイルを行わないため、コンパイル言語と比較してエラーに気づきづらいです。

演習の成果

  • 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

第4回はDOMを用いたWebページの編集のしかたを学びました。
DOMを使えばHTMLやSCCをプログラムで編集できるので、次の回で習うイベント処理と合わせればWebページに動的処理を実現できます。基本的にDOMを使う上では、HTMLファイルで各要素にidやclassを設定します。
DOMはWebページを変更してしまうため、DOMを実装する際はエディターでJSファイルを直接編集せずに、先にWeb開発ツールを用いて動作確認をしたほうがいい。

演習の成果

  • 演習4-2~4-5: DOMプログラミング

第5回 イベント処理

第5回はWebページのイベント処理について学びました。
イベント処理と第4回で学んだDOMを用いることで動きのあるWebページを作成することができます。
イベント処理はWebページの要素(ボタンやセルなど)にイベントハンドラを登録することで実現します。イベントハンドラの名前はそのイベントハンドラの処理がわかるような名前にしたほうがわかりやすいです。要素ごとにイベントは様々で、実現したい機能に合わせてイベントを選択します。

工夫点, 苦心点, 強調点, 感想

プロジェクト課題ついて書ききれなかったことをここに書きます。
世界の天気を実施する上で、Webページに世界地図を表示しましたが、各地点の位置を単純にpxで指定すると、Webページのウィンドウサイズを変更したときにずれてしまうので、そうならないようにJSで各地点の位置を修正する関数を作成し、ウィンドウサイズが変更されても世界地図と各地点の位置関係が変わらないようにしています。その際に、各地点の位置を決める定数を格納したリストを作ることで、簡単かつわかりやすく各地点の位置調整ができるようになっています。さらに、位置の修正だけでなく、位置を示す点のサイズやフォントサイズ、アイコンサイズ、ボーダーラインの幅など、さまざまな要素の大きさ、デザインをウィンドウサイズが変更されても相対的な関係は変わらず、Webページを拡大・縮小するようにしました。
また、第6回でAjax通信について学びましたが、プロジェクト課題では、最初にWebページを読み込む際に、全ての地点の天気データをサーバーから入手し、天気アイコンを各地点に表示しているので、Ajax通信はその際に一通り行っているわけです。そのため、以降通信して得られるデータは一度入手したことがあるデータなので、いっそのこと最初の通信で得たデータをプログラム内で保存して使いまわせば、最低限の通信で済ませられ、Webページの動作を早くなると思い、Webページ読み込み時の一斉通信のみ行っています。
また、Webページの各地点の要素とデータの結び付けは、都市名と都市IDのペアをもつリストを作成し、各要素のIDを都市名にすることで行っています。そうすることで、リストを介して要素のIDから都市ID、都市IDから要素のIDと相互に変換することができます。
また、天気の詳細をポップアップウィンドウ形式にすることでわかりやすく情報を表示していたりと、GUIについてはわかりやすさ、見やすさにこだわりました。
他にも、ある程度プログラムが大きくなるとWebページの動作が表面上だけではわかりずらく、エラーが発生した際には修正が困難なので、Webページの動作一つ一つをログに出力するようにして、今Webページが何をしているのか、Webページの動作の可視化と、どこでどのタイミングでエラーが発生しているかわかりやすくしています。個人的にはログの書き出しは重要だと思っていて、ある箇所のプログラムを変更したときに自分では関係ないと思っていた箇所がエラーを出すことがあって、そういうときにログがないとWebページが何をしてたらエラーを出したのか自分の頭で探り出すのは難しいので、ログ出力によってどこでエラーが出たのかいち早く理解できるのは大きいと思います。しかも、ログはそのときの変数の値も出力できるので、エラーが起きた時の状況をより詳しく知ることができます。
最後に、Webコンテンツ作成の感想ですが、Webページの作成は前から興味のあったことなので、かなりのめり込んで取り組めたと思います。授業の取り組み方も自由度が高くてよかったです。プロジェクト課題なんかはいい難易度かつ創造性のあるお題だと思うし、自由に作り込めるのはよかったです。自分たちB組は期末試験が全て終わってから1週間あったので時間に余裕があったのは大きいと思います。講義資料がわかりやすかったので、個人的には特に躓くこともなく授業を進められたかなと思います。かなりWebページ制作について詳しくなれたと思うのでよかったです。
