毎回の実験で作成したページ

第1回 WWW と HTML

第1回はWebページとWWWについてと、HTMLを用いてWebページの構造を記述する基本的な方法を学びました。
WebページはHTML, CSS, JavaScript, メディアの4つの部品からなり、WWWによってサーバからWebブラウザにWebページの部品が送られます。送られた部品をWebブラウザが組み立て、Webページを表示します。
HTMLはWebページの構造を記述する言語で、さまざまな要素でつくられます。要素には大きく分けてブロック要素とインライン要素があります。ブロック要素は基本的な構造を決める要素で、インライン要素はブロック要素を修飾してより細かく構造を決める要素です。

演習の成果

  • 演習1-3: HTML文書の作成
  • 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回はCSSを用いてWebページの装飾を記述する基本的な方法を学びました。
CSSはHTMLファイルに適用することでWebページの要素を装飾することができます。今回は、外部スタイルシートのファイルを指定する方法で適応しました。CSSにはさまざまなプロパティがあり、各プロパティに応じたプロパティ値を設定してWebページを装飾します。

演習の成果

  • 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

第3回はJavaScriptを用いて動きのあるWebページを実現するための基礎的なことを学びました。
JavaScriptはCSSと同様にHTMLファイルに適用することでWebページに’動き’を追加することができます。今回は、CSSと同様に外部指定でHTMLファイルに適用しました。その際にdefer属性を指定しないとプログラムが正しく動作しないことがあります。
Javaと構文は似ていますが、Javaとは違いインタプリタ言語で1行ずつ読み取りながら実行されます。コンパイルを行わないため、コンパイル言語と比較してエラーに気づきづらいです。

演習の成果

  • 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

第4回はDOMを用いたWebページの編集のしかたを学びました。
DOMを使えばHTMLやSCCをプログラムで編集できるので、次の回で習うイベント処理と合わせればWebページに動的処理を実現できます。基本的にDOMを使う上では、HTMLファイルで各要素にidやclassを設定します。
DOMはWebページを変更してしまうため、DOMを実装する際はエディターでJSファイルを直接編集せずに、先にWeb開発ツールを用いて動作確認をしたほうがいい。

演習の成果

  • 演習4-2~4-5: DOMプログラミング

第5回 イベント処理

第5回はWebページのイベント処理について学びました。
イベント処理と第4回で学んだDOMを用いることで動きのあるWebページを作成することができます。
イベント処理はWebページの要素(ボタンやセルなど)にイベントハンドラを登録することで実現します。イベントハンドラの名前はそのイベントハンドラの処理がわかるような名前にしたほうがわかりやすいです。要素ごとにイベントは様々で、実現したい機能に合わせてイベントを選択します。
